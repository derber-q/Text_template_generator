# Text Template Generator

Локальное desktop-приложение на Python для генерации одного итогового `.docx`-документа из Word-шаблона. Пользователь выбирает шаблон, задает количество копий, параметры номера и путь сохранения, а программа заменяет плейсхолдер `[№]` на уникальные номера.

## Возможности

- Выбор `.docx`-шаблона через графический интерфейс.
- Генерация заданного количества копий шаблона в одном итоговом `.docx`.
- Замена плейсхолдера `[№]` в обычных абзацах, таблицах, headers и footers.
- Генерация номеров формата `№<статическая_часть><случайные_цифры>`.
- Проверка уникальности случайных цифровых частей.
- Необязательный `.txt`-файл со списком запрещенных цифровых комбинаций.
- Необязательное сохранение использованных случайных комбинаций в `.txt`.
- Выбор системного шрифта и размера только для вставленного номера.
- Разрыв страницы между копиями, чтобы каждая новая копия начиналась с новой страницы.
- Запуск в Windows без видимого окна командной строки через `start.pyw`.
- Локальная иконка окна приложения из `assets/app_icon.png`.
- Окно прогресса при создании документа, чтобы видеть ход генерации.
- Понятные сообщения об ошибках через окно приложения.

## Структура проекта

- `main.py` — точка входа приложения.
- `start.pyw` — Windows-запуск без видимого окна командной строки.
- `assets/app_icon.png` — иконка окна приложения, сохраненная внутри проекта.
- `assets/README.md` — описание источника и назначения иконки.
- `gui.py` — окно tkinter, подсказки и обработка действий пользователя.
- `docx_processor.py` — чтение шаблона, замена `[№]`, сборка итогового `.docx`.
- `number_generator.py` — генерация уникальных номеров и чтение запрещенных комбинаций.
- `validators.py` — проверка пользовательских данных.
- `permanent_forbidden_combinations.txt` — постоянный список запрещенных случайных комбинаций.
- `tests/` — автоматические тесты генерации, валидации и обработки DOCX.

## Требования

- Windows, macOS или Linux с графической средой.
- Python 3.10 или новее.
- Установленный модуль `tkinter`. В стандартной установке Python для Windows он обычно уже есть.
- Пакеты из `requirements.txt`: `python-docx`, `docxcompose`, `pytest`.

Проверить Python в Windows можно так:

```powershell
py --version
```

Ожидаемый результат: версия Python, например `Python 3.14.3`.

## Как подготовить DOCX-шаблон

1. Откройте Microsoft Word или совместимый редактор.
2. Создайте обычный `.docx`-документ.
3. В том месте, где должен появиться номер, напишите точный плейсхолдер:

```text
[№]
```

4. Сохраните файл в формате `.docx`.

Пример текста в шаблоне:

```text
Договор [№] от 01.01.2026
```

После генерации это может стать:

```text
Договор №ABC-483920 от 01.01.2026
```

Важно: в шаблоне должны быть квадратные скобки. Программа ищет именно `[№]`, а не просто `№`.

## Как пользоваться приложением

### Шаг 1

1. Нажмите `Выбрать .docx`.
2. Выберите подготовленный Word-шаблон.
3. В поле `Количество копий` укажите, сколько копий шаблона нужно добавить в итоговый документ.
4. Выберите шрифт для номера.
5. Укажите размер шрифта.
6. Нажмите `Далее`.

Если в шаблоне нет `[№]`, приложение покажет ошибку и не перейдет к следующему шагу.

### Шаг 2

1. В поле `Постоянная часть без «№»` введите постоянную часть номера, например `ABC-`.
2. В поле `Количество случайных цифр` введите длину случайной части, например `6`.
3. При необходимости выберите `.txt`-файл запрещенных комбинаций. Он будет объединен с постоянными запретами из `permanent_forbidden_combinations.txt`.
4. Нажмите `Куда сохранить .docx` и выберите место сохранения итогового файла.
5. Если нужно сохранить использованные случайные цифровые части для будущих исключений, включите флаг `Создать .txt со сгенерированными цифровыми комбинациями`.
6. Нажмите `Создать DOCX`.
7. Дождитесь завершения в окне прогресса. Сначала будет отображаться создание копий, затем сохранение файла.
8. После фактического создания итогового документа появится окно `Готово` с путем к результату.

Символ `№` вводить не нужно. Программа добавляет его автоматически.

Если включено создание `.txt` со сгенерированными комбинациями, программа создаст файл рядом с итоговым документом. Например, для `generated_document.docx` будет создан `generated_document_generated_combinations.txt`. Внутри будут только случайные цифровые части через запятую без пробелов, например:

```text
483920,102938,000381
```

Этот файл можно позже выбрать как пользовательский список запрещенных комбинаций.

## Формат TXT-файла запрещенных комбинаций

В корне проекта есть постоянный файл `permanent_forbidden_combinations.txt`. Программа читает его при каждой генерации, даже если пользователь не выбрал дополнительный `.txt`-файл.

Пользовательский TXT-файл необязателен. Если он выбран, программа объединяет комбинации из пользовательского файла и `permanent_forbidden_combinations.txt`. Дубликаты между файлами учитываются как одна запрещенная комбинация.

Правильный формат:

```text
123456,984301,000381,555777
```

Правила:

- обычный `.txt`-файл;
- комбинации разделены запятыми;
- пробелов между комбинациями нет;
- каждая комбинация состоит только из цифр;
- длина каждой комбинации совпадает с полем `Количество случайных цифр`;
- дубликаты учитываются как одна запрещенная комбинация.

В репозитории `permanent_forbidden_combinations.txt` поставляется пустым. Если постоянных запретов пока нет, оставьте его пустым. Не добавляйте в него комментарии или поясняющий текст: файл должен содержать только цифры и запятые.

Если указано `6` случайных цифр, комбинация `12345` будет ошибкой, потому что в ней только 5 цифр.

Если постоянный файл содержит 4-значные комбинации, а в приложении выбрано `6` случайных цифр, программа тоже остановится с ошибкой длины. Все постоянные и пользовательские исключения должны иметь ту же длину, что указана в поле `Количество случайных цифр`.

## Проверка работоспособности

После установки зависимостей запустите тесты:

```powershell
py -m pytest
```

Ожидаемый результат: все тесты завершатся статусом `passed`.

Дополнительная ручная проверка:

1. Создайте простой `.docx` с текстом `Договор [№] от 01.01.2026`.
2. Запустите приложение командой `py start.pyw`.
3. Выберите этот шаблон.
4. Укажите 2 копии, шрифт и размер.
5. На втором шаге введите постоянную часть `ABC-` и 6 случайных цифр.
6. Сохраните итоговый файл.
7. Откройте итоговый `.docx` и проверьте, что в документе две копии, а `[№]` заменен на разные номера.

Проверка исключений:

1. Создайте временный `.txt` со строкой `000001,000002`.
2. В приложении укажите `6` случайных цифр и выберите этот `.txt` на втором шаге.
3. Сгенерируйте несколько копий.
4. Убедитесь, что в итоговом документе нет случайных частей `000001` и `000002`.
5. При включенном флаге создания TXT проверьте соседний файл `*_generated_combinations.txt`: внутри не должно быть повторов.

## Типовые ошибки

- `В выбранном шаблоне не найден плейсхолдер [№].`  
  Проверьте, что в `.docx` есть точный текст `[№]` с квадратными скобками.

- `Не вводите символ «№»: программа добавит его автоматически.`  
  Удалите `№` из поля постоянной части номера.

- `Длина каждой запрещенной комбинации должна совпадать с количеством случайных цифр.`  
  Исправьте `.txt`-файл или измените количество случайных цифр в приложении.

- `Невозможно создать нужное количество уникальных номеров.`  
  Увеличьте количество случайных цифр, уменьшите количество копий или сократите список запрещенных комбинаций.

- `Итоговый файл должен иметь расширение .docx.`  
  Выберите путь сохранения с расширением `.docx`.
